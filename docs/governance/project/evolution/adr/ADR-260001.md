# ADR-260001: Use React, TypeScript, and Vite for the timer app

Status: Accepted
Date: 2026-05-13
Owners: repository maintainer

## Context

Timplo is a static web app that must run on GitHub Pages, support mobile tap interactions, and stay simple enough to maintain without a backend.

## Decision

Build the app with React and TypeScript on Vite, keep timer state in browser memory with localStorage persistence, and deploy the built static files with GitHub Actions to GitHub Pages.

## Alternatives Considered

Formal alternatives were not minuted at decision time (v1). Implied by the constraints and effectively set aside:

- Option A: Plain HTML/JS/CSS with no framework — lower build complexity, but weaker component structure and state handling for the multi-timer UI.
- Option B: A backend-backed app (server-rendered or API-served state) — rejected by the "static hosting, no backend" constraint.

## Consequences

- Positive: fast local development, small deploy artifacts, and timer logic isolated in a small shared helper module (`src/timer.ts`).
- Negative: introduces a build step and a framework dependency for an otherwise small app.
- Risks: GitHub Pages base-path misconfiguration can break asset loading; mitigated by Vite's relative base path.

## Verification

`npm run build` produces static output that loads and runs on GitHub Pages; the critical in-browser timer flows work end-to-end from the built site.

## Rollback

This is a foundational stack decision. Reversal means re-scaffolding on a different stack; day-to-day changes remain reversible by reverting the offending commit and redeploying the prior successful build.
