# Requirement: REQ-QR-260003 Browser-local persistence

- Requirement ID: REQ-QR-260003
- Title: Browser-local persistence
- Class: QR
- Status: Active
- Owner: repository maintainer

## Statement

The application must preserve timer state locally in the browser so that timers survive a refresh on the same device.

## Rationale

Local persistence means an accidental refresh does not lose configured timers or in-progress countdowns, without needing an account or backend.

## Acceptance Criteria (behavioral)

- Timer state is restored after page reload.
- Persistence does not require a backend service.
- Corrupted stored data falls back to a safe default state.
