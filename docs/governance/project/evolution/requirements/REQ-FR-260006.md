# Requirement: REQ-FR-260006 Two-view screen structure

- Requirement ID: REQ-FR-260006
- Title: Two-view screen structure
- Class: FR
- Status: Active
- Owner: repository maintainer

## Statement

The application must use two primary views: a timer library view for managing timers and a selected timer view for running one timer at a larger size.

## Rationale

Two views separate management (library) from focused running (stage), matching the two distinct things users do with the app.

## Acceptance Criteria (behavioral)

- The timer library view is the default screen.
- The timer library view allows add, select, delete, and edit actions.
- The selected timer view shows one enlarged timer and a back action to return to the timer library.
- The selected timer view does not expose add, edit, or delete actions.
- The selected timer view exposes a reset action below the enlarged timer.
