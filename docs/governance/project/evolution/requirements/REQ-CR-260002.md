# Requirement: REQ-CR-260002 Timer limits and naming constraints

- Requirement ID: REQ-CR-260002
- Title: Timer limits and naming constraints
- Class: CR
- Status: Active
- Owner: repository maintainer

## Statement

The application must limit the timer collection and timer inputs to the visual range supported by the product.

## Rationale

Bounding timer count, duration range, and name length keeps the UI legible and the interaction simple, and matches the fixed `MM:SS` display.

## Acceptance Criteria (behavioral)

- The app allows at most 10 timers.
- A timer duration must be greater than `00:00`.
- A timer duration must not exceed `59:59`.
- A timer name must be concise and must not exceed 15 characters.
