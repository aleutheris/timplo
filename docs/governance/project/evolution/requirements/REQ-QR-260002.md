# Requirement: REQ-QR-260002 Mobile-friendly web experience

- Requirement ID: REQ-QR-260002
- Title: Mobile-friendly web experience
- Class: QR
- Status: Active
- Owner: repository maintainer

## Statement

The application must provide a responsive browser experience that works well on desktop and mobile devices, including touch interaction on phones and tablets.

## Rationale

Timers are frequently used on phones, so responsive layout and touch-equivalent interaction are essential to the core use case.

## Acceptance Criteria (behavioral)

- The layout adapts to smaller screens.
- Tap input behaves like click input for timer control.
- The active timer remains usable without requiring a keyboard.
