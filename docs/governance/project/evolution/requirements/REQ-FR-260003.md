# Requirement: REQ-FR-260003 Active timer switching and control

- Requirement ID: REQ-FR-260003
- Title: Active timer switching and control
- Class: FR
- Status: Active
- Owner: repository maintainer

## Statement

The application must let the user select one timer into a dedicated selected timer view with an enlarged square countdown area, and only the selected timer may run.

## Rationale

A single enlarged active view keeps interaction focused (especially on mobile); allowing only the selected timer to run keeps behavior predictable and switching instant.

## Acceptance Criteria (behavioral)

- Selecting a timer immediately shows it in the active countdown area.
- On phones, the enlarged timer occupies roughly half of the visible screen height.
- On larger screens, the selected timer is centered and significantly larger than timers in the library view.
- Selecting a timer does not start the countdown automatically.
- Clicking or tapping the active countdown starts, stops, or resumes the selected timer.
- If the user selects another timer, the previously selected timer stops running and keeps its remaining time.
- The same interaction works on desktop and mobile browsers.
