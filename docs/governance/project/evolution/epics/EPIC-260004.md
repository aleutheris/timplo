# Epic: EPIC-260004 Persistence, responsiveness, and deployment

State is tracked in `epic-index.md`; task states live in the inline checklist below. This epic
is **frozen** (Done) — retained for history per `artifact-model.md`.

- Epic ID: EPIC-260004
- Title: Persistence, responsiveness, and deployment
- Persona/Actor: someone using Timplo across devices and sessions
- Value statement: For a user on desktop or mobile, deliver local persistence, responsive layout, and static deployment so timers survive refreshes and the app runs well on GitHub Pages.
- Context: cross-cutting concerns that make the app usable and shippable beyond the two views.
- Problem statement: without persistence a refresh loses timers; without responsive layout mobile use suffers; without deployment the app cannot ship.
- Expected outcome: timer state persists in localStorage and restores after reload (including the selected timer), the layout adapts to mobile, and a GitHub Actions workflow publishes the static build.
- Success metric(s): timers and selection survive a refresh on the same device; the app is usable on phones; the deploy workflow publishes `dist`.
- Non-goals: new timer features (covered by EPIC-260002/003).
- Constraints: static hosting only, no backend (REQ-CR-260001, REQ-OR-260001).
- Dependencies: EPIC-260002, EPIC-260003 (a working app to persist and deploy).
- Risks/assumptions: corrupted stored data must fall back safely; GitHub Pages asset paths must resolve (relative base).
- Target window: third delivery slice.
- Validation approach: reload-restore checks, mobile responsiveness checks, and a successful GitHub Pages deploy; `npm run build`.

## Ideation Bullets

- [x] What persists and how? → timer definitions and state in localStorage under `timplo.timer-state.v1`.
- [x] What restores after refresh? → timers and the selected-timer context on the same device.
- [x] How is corrupt stored data handled? → fall back to a safe default state.
- [x] How does the app deploy? → GitHub Actions builds and publishes `dist` to GitHub Pages; Vite uses a relative base path.
